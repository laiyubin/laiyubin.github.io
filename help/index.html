Help

The SEP Book is for team members and by team members.

  • It’s the place to record SEP architecture and software design details, and everything an SEP team member need to know to start his/her work.
  • The book is continuous maintained by SEP team members. Everybody is encouraged to contribute to it to add new information or fix error.

The SEP Book is written in Markdown format and is generated by Jekyll, a static site generator.

Reading

Sections of the book

On top of SEP book there is a navigation bar, from which you can go to different sections.

There are four major sections in SEP book.

  • DESIGN contains architecture and software design details of SEP.

  • DEVELOPMENT contains information that an SEP team member need to know to start his/her daily work, for example: environment setup, guidelines, processes.

  • PROJECT contains project management information.

  • TOPICS are individual topics which doesn’t fit into previous sections. Topics are orginalized in blog style.

Writing

Syntax

SEP book is written in Markdown with some extensions.

Markdown basic syntax

Markdown is easy to learn by example.

Paragraph

Paragraphs are delimited by empty line. Line break is ignored in markdown.

Text modifier

Surround the text with either * or _ to emphasise. Strong emphasis can be done by doubling the delimiters. 

This is *emphasized*. Use two asterisks for **strong emphasis**.

It’s emphasized. Use two asterisks for strong emphasis.

Surround the text with ` to make it displayed in inline code style.

Hypherlink

[text](url)

Don’t include host part in URL for link to page inside SEP book. Example: /design/service-discovery/

Image

{% img ./class.png %}
{% img left ./class.png %}
{% img right ./class.png %}

Header

# H1 header
## H2 header
### H3 header
#### H4 header
##### H5 header
###### H6 header

Blockquote

A blockquote is a paragraph started using the >.

> Stay hungry, stay foolish.

is displayed as:

Stay hungry, stay foolish.

Code block

“Fence” a code block by ~~~:

~~~ ruby
def what?
  42
end
~~~

The language name is optional. There is syntax highlighting for supported languages.

List

Start by *, + or -. Indent sub item by 2 spaces. 

+ Item 1
  + Item 1.1
    + Item 1.1.1
- Item 2

Table

| No.   | Name    | Status  |
|------:|:--------|:-------:|
|1      | Alaph   | done    |
|2      | Beta    | ongoing |
|10000  | Release | n/a     |

The above markdown is rendered as below. Be noticed of the alignment.

No. Name Status
1 Alaph Done
20 Beta Ongoing
100 Release N/A

Definition list

foo
: Explaination of `foo`. 

bar
: Explaination of `bar`. 
  Other block-level format still can be used inside definition list, with indent, like:
  
  * item 1
  * item 2

  > quote

Note

Put inline attribute list (IAL) at the next line of a text block to display the block in different note styles.

The quick brown fox jumps over the lazy dog.
{: .note }

This is a note: {: .note }

This is an information: {: .note.info }

This is a warning: {: .note.warning }

This is about unrealeased feature: {: .note.unreleased }

Comment

Use comment marker to mark todo item on the article or review comment.

{: .comment }

Markdown advanced syntax

The markdown processor used in SEP book generating is Kramdown, which supports some additional syntax beyond starndard markdown. Detail can be found in kramdown’s site.

UML

It’s easy to embed UML diagrams into SEP book. Just wrap the diagram text in PlantUML format in plantuml block.

{% plantuml %}
Alice -> Bob: Hello, Bob
Alice <-- Bob: Hello, Alice
{% endplantuml %}

Flowchart

The activity diagram supported by Plantuml can be used as flowchart.

{% plantuml %}
start

if (graphviz installed?) then (yes)
  :process all diagrams;
else (no)
  :process only
  __sequence__ and __activity__ diagrams;
endif

stop
{% endplantuml %}

Graphviz

Graphviz DOT language is supported in SEP book via plantuml plugin. DOT is a plain text graph description lanuage, which is suitable to represent structural information as diagrams of abstract graphs and networks.

{% plantuml %}
@startdot
digraph DI {
  subgraph cluster1 {
    label="Spring context";
    pojo3 -> pojo1;
    pojo3 -> pojo2;
  }
  subgraph cluster2 {
    label="CDI context";
    ejb -> bean1;
  }
  ejb -> pojo3;
}
@enddot
{% endplantuml %}

Use @startdot and @enddot to surround the DOT language text. Details of DOT language please refer to online reference and user guides on graphviz.org.

Block diagrams

Block diagram can be generated from ASCII art. Please check ditaa for syntax. To draw the source ASCII art, a nice online service asciiflow can be used.

{% plantuml %}
@startditaa
+--------+   +-------+    +-------+
|cAAA    +---+ ditaa +--> |       |
|  Text  |   +-------+    |diagram|
|Document|   |!magic!|    |       |
|     {d}|   |       |    |       |
+---+----+   +-------+    +-------+
    :                         ^
    |       Lots of work      |
    +-------------------------+
@endditaa
{% endplantuml %}

Write locally

The book is a Git repository: ssh://<user_id>@gerritforge.lmera.ericsson.se:29418/sepbook. Clone it to your local disk to modify.

You can use any text editor to write Markdown, or choose an editor with Markdown support. MarkdownPad is one of the Markdown editor for Windows. Mou is one of the Markdown editors on Mac OS X. ReText is one of the Markdown editors on Linux.

Files/directories you need to know:

File/Dir Description
design Pages of DESIGN section. Put attachments into a sub-directory with the same name of the markdown file.
dev Pages of DEVELOPMENT section. Put attachments into a sub-directory with the same name of the markdown file.
project Pages of PROJECT section. Put attachments into a sub-directory with the same name of the markdown file.
_posts Posts of TOPICS section. The naming convention of these files is important, and must follow the format: YEAR-MONTH-DAY-title.md.
_data/docs/*.yml Defines the navigation menu at the right side.
_bin Stores tools used in generation.
_site This is where the generated site is placed.

Add new page

Create page file

Create a markdown file under design, dev or project directory. The file name is same as the last section of permalink (URL).

The file must have a header at the beginning:

---
layout: docs
title: Page Title
permalink: /design/chapter_b/
---

The file name of above example is chapter_b.md, it’s URL will be http://<host>/design/chapter_b/.

Register page to navigation menu

The new page will not be shown in the navigation sidebar at the right until being registered in the yml file under _data/docs/. The file name design, dev and project are corresponding to the sections on top navigation bar.

Groups
Group the pages in sidebar. New group can be added if needed.
Pages
List permalink of the pages.

To register the page chapter_b.md in above example into a group called “New Group” in DESIGN section, update the design.yml like: 

1
2
3
4
5

  - title: New Group
    docs:
    - /design/chapter_a/
    - /design/chapter_b/
    - /design/chapter_c/

Storing assets

The page may has assets, e.g. image files, PowerPoint slides, they should be stored in a folder in the same name as the markdown file (without suffix).

design/chapter_b.md
design/chapter_b/pic.jpg
design/chapter_b/qs.pptx

The asset can be referred by related hyperlink. For example, to embbed the pic.jpg in markdown, use {% img ./pic.jpg %}.

Add new post

Create post file

Create a markdown file under _posts directory. The file need to be named according to the format: YYYY-MM-DD-title.md. The date is current date that you publish the post. Do not change the file name when you update the post, if you did so it breaks existing link.

Unlike pages, it isn’t required to register post into navigation menu. All posts are shown under TOPICS section ordered by date.

The file must have a header at the beginning:

---
layout: post
title:  "From EJB 2.1 to EJB 3.x"
---

Storing assets

_posts/2014-07-29-test.md
topics/2014-07-29/pic.jpg

Post asset should be put in folder topics/YYYY-MM-DD (create if it doesn’t exist). The date is the same as that in your post file name. In the post, refer to the asset by related hyperlink {% img ./pic.jpg %}.

Preview

Windows

To view final visualization of the book, you need to use Jekyll to generate it. An all-in-one Jekyll executable for Windows is already included in the SEPbook repo. All you need to do is run _bin/preview.bat then open http://localhost:4000/ in your browser. While preview is running, it monitors any file change and re-generate on-the-fly.

Java Runtime Environment is required to be installed before running preview. Graphviz is required to be installed before running preview. Download the msi installer and install it in default directory. PlantUML looks for Graphviz dot in either c:\Program Files\Graphviz*\dot.exe or c:\Program Files (x86)\Graphviz*\dot.exe. If Graphviz isn’t installed in that directory, need to set environment variable GRAPHVIZ_DOT to specify the full path name to dot.exe.

Graphviz-2.38.msi installer has bug, it always install into c:\bin\dot.exe in spite of the installation path setting. So have to set environment variable GRAPHVIZ_DOT to c:\bin\dot.exe.

Mac OS X

Install Graphviz using Homebrew.

$ brew install graphviz

Install Jekyll using gem.

$ sudo gem install jekyll

Go to the project home directory and run jekyll to start the local server.

$ jekyll serve

While preview is running, it monitors any file change and re-generate on-the-fly.

OpenSUSE

Install ruby and execjs

$ sudo zypper ruby ruby-devel execjs

Install Jekyll using gem.

$ sudo gem install jekyll

Go to the project home directory and run jekyll to start the local server.

$ jekyll serve

Publish

The SEP book is built and published by a CI job. Once there is change on the git repository, the updated version will be published on http://SEP-book.sh.cn.ao.ericsson.se/.

The CI job also generates Git commit stats report with GitStats. Link to the report can be found at page footer.
